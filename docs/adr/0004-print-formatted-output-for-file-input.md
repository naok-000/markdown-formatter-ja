# 0004. CLIのファイル指定時も整形結果を標準出力する

## Status

Accepted

## Context

markdown-formatter-ja は，Markdownの整形結果を他のコマンドやエディタ連携に組み込みやすいCLIにしたい．

以前は，ファイルパスを指定し，`--write` を指定しない場合は，整形前後の差分を標準出力へ表示する方針だった．
しかし，現時点ではdiff表示は必須ではない．
また，編集後のMarkdownそのものが標準出力される方が，パイプやリダイレクトに組み込みやすい．

diff表示は，将来必要になった時点で `--diff` オプションなどとして追加できる．

## Decision

ファイルパスを指定し，`--write` を指定しない場合も，整形結果を標準出力へ書く．

ファイルを直接書き換える場合は，明示的に `--write` を指定する．

現時点ではdiff表示を実装しない．

## Consequences

- 標準入力とファイル入力で，標準出力の内容が整形結果に揃う
- パイプやリダイレクトに組み込みやすい
- 変更内容の確認にdiffが必要な場合は，外部コマンドを使う
- 将来必要になれば，`--diff` オプションとして追加できる
