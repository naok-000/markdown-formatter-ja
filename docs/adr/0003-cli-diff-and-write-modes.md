# 0003. CLIのファイル指定時は差分表示をデフォルトにする

## Status

Accepted

## Context

markdown-formatter-ja は，Markdownファイルのraw textを読みやすく整形するCLIである．
CLIには，標準入力から整形結果を標準出力へ出す使い方と，ファイルを指定して整形する使い方がある．

ファイル指定時に常に標準出力へ整形結果を出すと，変更内容の確認には別途diffが必要になる．
一方，デフォルトでファイルを書き換えると，利用者が意図せずファイルを変更する可能性がある．

また，`--help` や引数バリデーションを自前実装で増やしていくと，CLI本体に本質でない分岐が増える．

## Decision

ファイルパスを指定し，`--write` を指定しない場合は，ファイルを書き換えずに整形前後の差分を標準出力へ表示する．

ファイルを直接書き換える場合は，明示的に `--write` を指定する．

標準入力から読む場合は，従来どおり整形結果を標準出力へ書く．

CLI引数解析には，自前実装ではなく `clap` などのcrateを使う．

## Consequences

- ファイル指定時に，変更内容を実行結果だけで確認できる
- ファイルの直接変更は明示的な操作になる
- 標準入力と標準出力を使うエディタ連携は維持できる
- 差分表示の実装が必要になる
- `--help` や不正な引数の扱いは，CLI引数解析crateの標準的な振る舞いに寄せられる
