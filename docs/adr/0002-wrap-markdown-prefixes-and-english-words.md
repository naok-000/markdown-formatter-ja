# 0002. Markdownの行頭マーカーと英単語を折り返し単位として扱う

## Status

Accepted

## Context

markdown-formatter-ja は，Markdownのraw textを読みやすく整形する．
そのため，指定幅で折り返した結果も，表示上の1行幅ができるだけ揃っている必要がある．

リスト項目では，本文の前に `- ` や `1. ` などのMarkdown記法が置かれる．
本文だけを指定幅で折り返すと，raw text上ではマーカー分だけ行幅が超過し，リストの行だけ長く見える．
ネストしたリストでは，インデントとマーカーが加わるため，このずれはさらに大きくなる．

また，英単語を文字単位で折り返すと，`markdow\nn` のように単語が壊れ，raw textとして読みにくくなる．
日本語の文章中にも英単語，識別子，ファイルパス，URL片が混在するため，これらを途中で分割しない方針が必要である．

## Decision

リスト項目を折り返すときは，行頭から本文開始位置までの表示幅を指定幅に含める．

- 箇条書きでは，`- ` の幅を行幅に含める
- 番号付き箇条書きでは，`1. ` や `10. ` など，実際のマーカー幅を行幅に含める
- ネストしたリストでは，インデントとマーカーの合計幅を行幅に含める
- リスト項目の継続行は本文開始位置に揃える
- 継続行のインデントも行幅に含める

たとえば `width = 10` の場合，`- ` の幅2を差し引き，本文は幅8で折り返す．

```markdown
- これは日本語の項目です
```

```markdown
- これは日
  本語の項
  目です
```

英単語は途中で改行しない．
ここでの英単語は，ASCII英数字，`_`，`-`，`.`，`/` の連続とする．
たとえば `foo_bar`，`foo-bar`，`example.com`，`path/to/file` は途中で分割しない．

英単語が指定幅を超える場合は，単語を分割せず，その行だけ行幅超過を許す．

## Consequences

- リスト項目の行幅が，raw text上の見た目に近い幅で揃う
- ネストしたリストほど本文に使える幅は短くなる
- 番号付き箇条書きでは，番号の桁数によって本文に使える幅が変わる
- 長い英単語，URL片，ファイルパスは行幅を超えることがある
- 英単語の範囲は単純な文字集合で判定し，形態素解析や自然言語処理は行わない
